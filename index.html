NIST Core Definition (SP 800-145)

"Cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction."

Source Document: NIST Special Publication 800-145 Official Release

Cloud Service Models

Toggle across execution layers to inspect the Shared Responsibility Model, mapping exactly how the distribution of infrastructure, virtualization, and code management shifts between cloud vendor and consumer.

Infrastructure as a Service (IaaS)

Provides raw compute, block storage, and software-defined networking primitives. Offers maximum architectural flexibility, leaving OS kernel tuning, routing rules, and data schemas entirely in your hands.

Provider Managed Infrastructure

Physical plant facilities, power distribution units, climate systems, optical networking arrays, bare-metal server chassis, and Type-1 Hypervisors.

User Managed Configuration

Guest Operating System distributions (patching, security configurations), file systems, network firewalls, container engines, middleware, and application code.

Platform as a Service (PaaS)

Removes the operational overhead of server patching, capacity provisioning, and OS maintenance. Abstracted runtimes let developers focus entirely on application logical components and database configurations.

Provider Managed Infrastructure

Automated OS patch lifecycles, memory allocation pooling, runtime compilation environments (Node.js, JVM, CLR), load balancer pools, and cluster management tools.

User Managed Configuration

Application source files, API routes, data persistence configurations, access keys, webhook pipelines, and fine-grained environment variables.

Software as a Service (SaaS)

A completely centralized deployment strategy where the application stack is entirely executed and maintained by a third-party vendor, consumed over the web via secure endpoints or specialized desktop thin clients.

Provider Managed Infrastructure

100% of the architecture including physical infrastructure, databases, UI rendering, scaling mechanisms, security patches, and application logical bugs.

User Managed Configuration

Granular RBAC (Role-Based Access Control) user seats, tenant preference parameters, notification policies, and custom third-party OAuth integrations.

Cloud Deployment Models

Structural isolation schemas defining ownership boundaries, multi-tenancy rules, compliance vectors, and physical hardware geographies.

Public Cloud Topology

Dynamic multi-tenant environments where massive physical server capacity is shared across independent customers. Sub-millisecond isolation is strictly maintained at the hypervisor and Software-Defined Networking (SDN) layers.

VMs vs Containers Architecture

Deep comparison of modern runtime execution. The foundational choice dictates computing efficiency, application delivery cadence, and infrastructure cost.

Virtual Machines (Hardware Abstraction)
  • Hypervisor Isolation: Uses a Type-1 or Type-2 hypervisor to partition raw CPU, memory, and PCIe lanes into independent segments. For deep architectural implementation details, read the Intel VT-x Specs.
  • Guest OS Burden: Each virtual instance runs a complete, heavy guest operating system kernel, requiring gigabytes of disk and RAM footprint alongside long boot sequences.
  • Hardware Security Boundary: Strong isolation verified at the CPU microcode level, making it highly secure for executing untrusted, isolated code execution loops.
Containers (OS Kernel Abstraction)
  • Kernel Space Sharing: Containers run directly as isolated processes on the host operating system kernel, eliminating hypervisor overhead and achieving near bare-metal operational execution.
  • Primitive Control Blocks: Uses standard Linux native namespaces (isolating PID, Mount, Net) and Control Groups (cgroups for memory/CPU capping). Study the engineering specifics at the Linux Kernel Archive.
  • Rapid Microservices Lifecycle: Instant boot-up speeds (milliseconds) and tiny image sizes (megabytes) enable agile immutable CI/CD software scaling loops.

The 5 Essential Characteristics

The five absolute architectural features that mathematically distinguish cloud infrastructures from standard traditional server colocation centers.

On-Demand Self-Service
Consumers provision storage, compute, and cluster routing configurations automatically through automated API control panels and infrastructure-as-code scripts, without needing manual human intervention from the provider's operations staff.
Broad Network Access
Capabilities are accessible over public or private networks through standardized protocols (HTTP, gRPC, WebSockets), enabling thin and thick clients alike (smartphones, enterprise laptops, headless IoT clusters) to consume services globally.
Multi-Tenant Resource Pooling
The provider's hardware assets are unified inside deep resource matrices to serve multiple consumers simultaneously. Physical processors, disk drives, and cache modules are dynamically mapped, reassigned, and isolated at runtime according to real-time consumer requests.
Rapid Elasticity & Scale
Services can be scaled horizontally or vertically automatically based on computational load profiles. To the user, provisioning feels completely infinite, allowing systems to scale from a single container node to a massive thousands-node cluster within minutes.
Measured Service Billing
Cloud resources are monitored transparently via metering telemetry algorithms at a high level of granularity (e.g., per gigabyte-hour, per million API invocations, or per compute vCPU minute). Resource consumption metrics can be mapped directly to chargeback reporting schemas.
